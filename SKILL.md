# 热点文案改写（学员版）

## 核心逻辑

**热点是钩子，业务是落脚点。**

```
热点事件（引流量）→ 巧妙转折（接业务）→ 自己的产品/服务（精准获客）
```

不是写一篇热点评论然后硬接广告，而是让观众看完觉得"这个热点跟我有关系，这个人能帮我解决问题"。

## 需要的信息

从系统记忆、用户画像、对话上下文中获取以下信息。**已有的直接用，缺的才问**：

1. **你是做什么的**：行业、具体产品/服务（如：火锅店、门窗厂、钢琴培训、财税咨询）
2. **你的客户是谁**：目标客群画像（如：装修业主、餐饮创业者、3-12岁孩子家长）
3. **你帮客户解决什么问题**：核心价值（如：省装修预算、学会一道招牌菜、孩子考级通过）
4. **你的差异化**：为什么选你不选别人（如：20年经验、自有工厂、通过率95%）

**严禁** 在没有确认用户业务信息的情况下直接写文案。写出来的文案必须和用户的具体业务强相关。

## 工作流程

### Step 1：热点拆解

收到热点后，内部完成分析（不需要全部外显）：

1. **事实核心**：这个热点到底发生了什么（不能搞错）
2. **公众情绪**：大家在气什么/惊什么/笑什么
3. **行业连接点**：这个热点跟用户的行业有什么关系？（这是最关键的一步）

**行业连接点的寻找方法**：

| 热点类型 | 连接思路 | 示例 |
|---------|---------|------|
| 消费维权类 | 行业信任问题 → 你怎么做的不一样 | 某品牌造假 → "做火锅10年，我的底料配方贴在墙上" |
| 职场/创业类 | 经营痛点共鸣 → 你的解法 | 大厂裁员 → "做实体的我们不怕被裁，怕的是没客人" |
| 社会热议类 | 价值观共鸣 → 你的经营态度 | 某明星塌房 → "做生意跟做人一样，口碑塌了就完了" |
| 政策/经济类 | 行业影响分析 → 你的应对方案 | 新税收政策 → "小微企业怎么合规省税，我研究了3天" |
| 民生/物价类 | 成本压力共情 → 你的真实成本 | 猪肉涨价 → "我给你算算一碗牛肉面的成本" |

### Step 2：确认切入角度（必须等用户选）

给出 2-3 个角度让用户选：

```
【热点】：xxx事件
【你的行业】：xxx
【连接点】：xxx

【切入角度】
1. xxx — 从xxx角度切入，适合引发xxx情绪
2. xxx — 从xxx角度切入，适合展示xxx优势
3. xxx — 从xxx角度切入，适合获取xxx类客户

你选哪个？选完我开始写。
```

**严禁** 跳过用户选择直接写。

### Step 2.5：查阅行业范例

在写文案之前，先读 `知识库/行业改写范例.md`，找到和用户行业最接近的范例，学习它的**转折写法**。

范例库展示了同一个热点在5个不同行业（餐饮/制造/美业/教培/汽修）的改写方式。核心学习点不是内容，是**转折的逻辑**：

> 这个热点暴露的问题，在我的行业也存在，但我是怎么做的不一样的。

如果范例库里没有完全匹配的行业，找商业逻辑最接近的（如：健身找美业、装修找制造、咨询找教培）。

### Step 3：撰写文案

按以下结构写，核心就是 **前面蹭热点，中间转业务**：

#### 结构骨架（5段式）

**① 3秒站队开场（约5%）**
- 直接对热点表态，不先铺背景
- 制造情绪锚点，让人停下来
- 示例："今天看到xxx这个事，我作为一个做了X年xxx的人，真的坐不住了。"

**② 热点还原 + 深度解读（约30%）**
- 用画面感还原事件（人物+动作+一句关键话）
- 给一个大多数人没想到的角度/真相
- 让观众觉得"看得真透"

**③ 巧妙转折 — 接到自己的行业（约20%）**

这是全篇最关键的一段。**转折必须自然，不能硬接。**

好的转折方式：
- **共性问题**："其实这个事背后的问题，在我们xxx行业一样存在。"
- **反面教材**："你别笑，做xxx的要是也这么干，结果一模一样。"
- **正面对标**："所以我做xxx这么多年，有一条铁律——xxx。"
- **客户共情**："我的客户经常问我同样的问题——xxx。"

**严禁** 的转折方式：
- ❌ "说到这里，顺便介绍一下我们的产品…"
- ❌ "好了，热点聊完了，接下来说说我的业务…"
- ❌ 突然插入毫无关联的业务介绍

**④ 业务价值输出（约30%）**
- 结合热点引出的问题，讲你怎么做的、为什么这么做
- 用真实案例/数字/成本账证明
- 让观众觉得"这个人确实懂"

**⑤ 金句收口 + 低压CTA（约15%）**
- 一句可被截图传播的结论
- 只做低压引导：关注 / 评论 / 私信领资料
- **严禁** 硬逼单（"最后X个名额""不买就亏了"）

#### 文案要求
- 字数：600-1200字（短视频口播，不是公众号长文）
- 口语化，像跟朋友聊天
- 短句优先，每段一个中心观点
- 热点占比 ≤ 50%，业务占比 ≥ 30%，收口占比约 20%

#### 语气风格

**从用户画像/系统记忆中读取用户的语气偏好。** 如果没有，默认：
- 真诚、接地气、不端着
- 像一个做了很多年的老手在跟你掏心窝子
- 可以犀利但不刻薄，可以直接但不粗暴

### Step 4：自检评分

| 维度 | 满分 | 关键问题 |
|------|------|---------|
| 热点准确性 | 20 | 事实有没有搞错？ |
| 转折自然度 | 25 | 从热点到业务的转折生不生硬？（最重要） |
| 业务相关性 | 25 | 文案和用户的具体业务强相关吗？ |
| 听感可读性 | 15 | 能不能直接拿来读稿？ |
| 风险边界 | 15 | 有没有造谣/夸大/恐吓/硬逼单？ |

- 总分 < 85 → 自动重写
- 转折自然度 < 20 → 必须重写（这是核心）
- 输出评分后直接给文案，不啰嗦

## 严禁事项

1. **严禁** 热点和业务硬接。转折必须自然，必须有逻辑连接点。
2. **严禁** 业务内容超过全文 40%。热点是主角，业务是配角。
3. **严禁** 散播焦虑和恐吓（"不做就死""再不行动就晚了"）。
4. **严禁** 编造热点事实或夸大事件。
5. **严禁** 硬逼单（"赶紧下单""最后X个名额"）。
6. **严禁** 在没确认用户业务的情况下写文案。
7. **严禁** 写成泛流量评论文。每篇必须落到用户的具体业务上。
